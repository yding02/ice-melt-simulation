Ice Melt Simulator

Abstract

Modeling of melting ice is a difficult task because the model needs to take into account the difference in melting rate between edges (a corner case of sorts) and faces as a result of the difference in surface area. As a result, the edges of a proper model seem to collapse, rounding out while the faces shrink at a slower rate. Our model renders an ice cube as a matrix of voxels. It then applies Euler’s method over the Thermodynamic Formula to calculate the next phase of an ice cube melting.

Technical Approach

Modeling (Cedric, Simon)

Our rendering is done in OpenGL. We used the library to create a grid of voxels to represent the pieces of the ice cube in our rendering, taking advantage of geometry shaders to accelerate the repeated generation of voxel cubes. We used a shader pipeline to determine colors at each position and the opacity. We also added diffuse normal shading to the voxel surfaces. The normals are generated in the geometry shader and passed to the vertex shader where the cosine weighting occurs with the light direction and color. Since we defined a light source, we added specular shading as well, using ice’s true specular value of .224. Although we explored modifying the base .dae files in Project 3 to render the ice using microfacet BSDFs, we ultimately decided that applying ray-tracing to ice was outside of the scope of our project.

Simulation (Yifeng)

Representation: In order to simulate an ice block, we split the block into smaller voxels. Each voxel has its own temperature and state, which helps simulate local temperatures on the ice block, as the ice block is not uniformly heated.  The ice block itself contains information about the dimensions of the ice cube and the number of voxels.

Thermodynamics: We apply the heat transfer formula,  to determine the amount of power that is transferred to each voxel. To do this, we calculated the area of contact between voxels and their temperature differences. We also used standard coefficients from Wikipedia. We assume that the ambient background is constantly applied to surface faces of the ice cube.

Updating: We use Euler’s method to calculate work done on each voxel for a certain timestep. Although this overestimates work done, the effects are not significant if we choose reasonable temperatures and timestep. Using the work, we calculate the change of temperature inside the voxel as well as its progress on melting using specific heat and fusion of enthalpy.

Optimization: After implementing the thermodynamics we realized the program ran very slowly. In order to speed up the program we made a key observation: only energy transferred from the background was significantly affecting voxel temperatures. Thus we only needed to consider surface voxels. We further made the observation that we can iteratively compute surface voxels from an initial set of surface voxels, since only neighbors of current surface voxels are candidates to become surface voxels in the next step. Additional Features (Cedric, Simon, Chris)  

We also created a dynamic camera to add different views of the scene, and we added listeners in GLFW to pause, unpause, and replay the simulation.

Results

The ice cube melts very nicely. We believe that the odd sort of rippling effect occurs on the cube because of how OpenGL renders the normals. Since only one normal gets assigned to each vertex, the lighting on some faces are distorted because normals are not facing the correct way. If we had more time, we would fix this by duplicating vertices so we could assign multiple normals to each vertex position. However, because of the extra storage this would require and the fact that the current lighting still looks relatively acceptable, we decided to keep this as is.
